Problem Statement

Create a version control account on GitHub and use Git commands to create a repository and push code to GitHub.

Objective

To understand GitHub account creation, Git version control workflow, repository creation, and pushing local project files to a remote GitHub repository.

Software Requirements

  • GitHub account
  • Git installed on the system
  • VS Code or any text editor
  • Internet connection

Theory

Git is a distributed version control system used to track changes in source code. GitHub is a cloud platform used to host Git repositories and collaborate on projects. By using Git and GitHub together, a developer can manage code versions, keep project history, and publish code online.

Steps Performed

  1. Create a GitHub account by signing up on https://github.com.
  2. Install Git on the local machine and verify it using the version command.
  3. Configure Git username and email.
  4. Create a new project folder and initialize it as a Git repository.
  5. Add project files to the staging area.
  6. Create the first commit.
  7. Create a new repository on GitHub.
  8. Link the local repository to the GitHub remote repository.
  9. Push the local project to GitHub.

Git Commands Used

git --version git config --global user.name "Atharv Avhad" git config --global user.email "avhadatharv587@gmail.com" mkdir assignment-2a cd assignment-2a git init git status git add . git commit -m "Add Assignment 2a project" git branch -M main git remote add origin https://github.com/AvhadAtharv/repository-name.git git push -u origin main

Explanation of Commands

  • git --version: checks whether Git is installed.
  • git config: sets user information for commits.
  • git init: initializes a new local Git repository.
  • git status: shows the status of files.
  • git add .: adds all files to the staging area.
  • git commit -m: saves a snapshot of staged files.
  • git branch -M main: renames the branch to main.
  • git remote add origin: links the local repository to GitHub.
  • git push -u origin main: uploads code to GitHub.

Sample Output

After successful execution, the project files become visible in the GitHub repository under the user account.

Conclusion

Thus, a GitHub account was used with Git commands to create a repository, manage version control locally, and push the project successfully to GitHub.
